proj open — open project in editor

Usage:
  proj open <ProjectName|basename> [options] [<editorName>]

Options:
  -r, --root     Open the project root (pa_* folder), not a worktree
  --wt <slug>    Open worktrees/<ProjectName>-<slug> (slug or full folder name)

Default: opens the main worktree worktrees/<ProjectName>-main (even if there is no .git yet).
-r and --wt cannot be used together.

Without <editorName>: if .projrc has `editor = <name>` and that name is configured via
`proj editor`, that template is used. Otherwise uses $EDITOR, then $VISUAL, then `code`.

Editor templates (proj editor / .projrc editor = …) support placeholders; see proj editor help.
Notably $dir is the opened folder, $root is the pa_* project root, and $rc_<key> reads .projrc
(e.g. $rc_workspace for a .code-workspace path).

VS Code workspace: add to .projrc, for example:
  workspace = MyApp.code-workspace
(path relative to project root, or absolute). Use a template such as:
  proj editor vscode-ws 'code $rc_workspace'
  proj set MyApp editor vscode-ws
(or pass vscode-ws on the command line: proj open MyApp vscode-ws)

Examples:
  proj open MyProject
  proj open MyProject -r
  proj open MyProject --wt StoreRefactor
  proj editor vim 'nvim $dir'
  proj open MyProject vim
