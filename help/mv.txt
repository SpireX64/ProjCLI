proj mv — rename under projects root

Usage:
  proj mv <fromBasename> <toBasename>
  proj mv <ProjectName>   Interactive rename (empty input keeps current field)

Paths are always relative to the configured projects root. For other moves use
the system mv command.

Tags are stored in each project’s .projrc (`tags`); change them with
`proj tag … -p`, not with mv.

When the ProjectName in the folder basename changes, directories under
`worktrees/<Project>-<slug>` are renamed to match, and git metadata is updated
(`git worktree repair` / `git worktree move`). Git 2.30+ is required if the
main worktree has a `.git` directory. If a command fails, check
`git worktree list` and try `git worktree repair` from the main worktree.
